JDC submission procedures##

Submission workflow/steps##

flowchart TB collect[Collect data via REDCAP,Qualtrics, or other method]; export[Export data in format of choice -- xlsx, spss, csv, etc]; summary[Fill out the template file\ncontaining subgroup summaries]; send[DASC tools create synthetic data from template] map[Map and transform variables and values to JDC ]; output[Output the JDC-transformed dataset to a TSV file]; submit[Submit the TSV within the JDC upload tool]; collect -- no individual-level\ndata collection --> summary --> send --> submit collect -- individual level\ndata collection --> export --> map --> output --> submit click map "https://rcg.bsd.uchicago.edu/gitlab/maarc/dasc" "Takes you to JDC utilities for creating submissions" click submit "https://jcoin.datacommons.io/submission#" "Takes you to submission page" click summary "https://uchicago.box.com/s/k0cu05u3lzyg6ca2owva5qpfqoj5xuwt" "Takes you to template -- contact XXX for access"

How to use the submission tool on JDC##

To use the JCOIN website for submission:

flowchart TB website[Go to jcoin.data.commons.io/submission or the "submit data" button on the JDC]; project[Scroll to your project under "List of projects"]; projectsubmit[Click "submit data" (this should take you to a link)]; upload[Click "upload file"]; find[Find the node (ie variable category) file]; website --> project --> projectsubmit --> upload --> find Do this for each variable category
flowchart TB decision_data_download[Is there a way to programmatically download data via an API or web scraping?]; data_download_manual[(NORC) Download data manually and move to directory with associated subdirectories]; data_download_auto[(NORC) Run code that creates and/or downloads all study files (API or from url if webscraping)]; data_upload[(NORC) Upload to AWS s3 bucket on heal external data using aws command line tool ] data_index[(CTDS) Run indexd on heal external to assign object ids to the new files]; is_adapter[Is there a metadata adapter?] is_adapter_config[In the configuration file, is there a correct set of mappings that currently uses that adapter?] get_api_mappings[Get correct mappings for set of studies to configuration file through researching API] make_api_mappings[Run code that adds additional studies of study API --> heal mappings and transformations for metadata and individual file properties] copy_config[Copy the new entries to the existing aggregate_configuration.json file (currently manually done to ensure it is easy to see revisions)] pr_qa[Pull Request to HEAL QA] pr_preprod[Pull Request to HEAL PREPROD] pr_prod[Pull Request to HEAL PROD] decision_data_download -- yes --> data_download_auto --> data_index decision_data_download -- no --> data_download_manual --> data_index data_index --> is_adapter -- yes --> is_adapter_config --> make_api_mappings data_index --> is_adapter -- no --> get_api_mappings --> make_api_mappings make_api_mappings --> copy_config -- to QA repository --> pr_qa -- copy to PREPROD --> pr_preprod --> copy to PROD --> pr_prod
